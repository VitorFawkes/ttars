Como a IA é transparente (não é caixa-preta)
  1. User pergunta (texto ou áudio). Se áudio, Whisper transcreve.
  2. IA interpreta e retorna JSON estruturado: {measure, groupBy, crossWith, filters, viz}.
  3. Exibe "Entendi assim" em destaque amarelo (visível, não escondido).
  4. Preenche o painel pivot automaticamente com esses valores.
  5. Executa e renderiza — user vê resultado + interpretação lado a lado.
  6. Se errou: user tweaka o painel pivot manualmente e reexecuta. O pivot é a fonte de verdade.
  7. Se quiser refinar: botão "💬 Refinar" abre follow-up ("agora só Planner João", "troca pra Ticket médio em vez de Receita") — multi-turn conversacional.
Arquitetura técnica (resumo honesto)

IA NÃO gera SQL direto (perigoso — alucinação, permissões, performance). Em vez disso: tool-use.

  • Tools expostos pra IA: as RPCs de analytics + descrição de parâmetros e dimensões válidas
  • LLM escolhe tool + preenche args (text-to-tool, não text-to-SQL)
  • Validação server-side: args são validados antes de executar RPC
  • RLS continua protegendo dado (IA não pode burlar permissão)
  • Fallback: se IA não tem confiança, retorna mensagem "Não entendi — monte no painel manualmente"

Pilhas candidatas: Claude/GPT com tool-use + Whisper pra áudio. Você já tem infra de IA (Luna, Julia, extração), então plumbing existe.

Trade-offs honestos pro escopo do v2

Ganhos

  • Barreira de uso zero (Vitor e gestores podem perguntar sem aprender pivot)
  • Áudio é mobile-friendly (perguntar "no carro" voltando de cliente)
  • Consistente com resto do CRM (Julia/Luna)
  • Pivot manual fica como fallback + power-user tool
  • Transparência (user sempre vê a interpretação)

Custos / riscos

  • +1 componente IA pra manter (prompt engineering, tool schemas, testes)
  • Custo por query (LLM + Whisper) — $0,01-0,05/pergunta, pequeno mas real
  • Latência (2-5s vs 0.5s do pivot manual)
  • Alucinação: precisa UX que deixa claro quando IA errou
  • Escopo v2 cresce — estimo +3-5 dias a mais que o Explorar manual puro
